# E-Commerce Website Testing Project

It is a manual testing project for an e-commerce web application (demo: https://demo.opencart.com).  
It demonstrates the Software Testing Life Cycle (STLC) — from requirement analysis to bug reporting.

## Project Overview
- Objective: Validate core e-commerce workflows: register, login, search, add to cart, checkout, payment flow, and UI/UX checks.
- Demo site: https://demo.opencart.com

## Artifacts
- `Documents/`
  - `Ecommerce_FRS.md` – Functional Requirement Specification
  - `Ecommerce_TestPlan.md` – Test Plan
  - `Ecommerce_TestCases.csv` – Test cases
  - `Ecommerce_RTM.csv` – Requirements Traceability Matrix
  - `Ecommerce_BugReport.csv` – Bug log
- `Jira_Screenshots/` – Jira issues and board screenshots

## Tools Used
- Jira (for project tracking)
- Excel (for test cases and RTM)

## How to use
1. Clone the repository.
2. Browse `Documents/` and open the CSV/MD files.
3. Run test cases on the demo site and update the `Ecommerce_TestCases.csv` and `Ecommerce_BugReport.csv`.
